# WebSmallGame 项目记忆归档

## 当前快照

`WebSmallGame` 是一个静态网页小游戏项目，主体是霓虹赛博风格的贪吃蛇游戏。
项目使用 p5.js 和原生 JavaScript 编写，没有包管理器、构建工具或本地服务依赖。

当前版本已经不只是单局小游戏，而是带有本地成长系统：

- 霓虹合约
- 成就墙
- 每日挑战
- 芯片奖励
- 各模式最高分

成长进度保存在浏览器 `localStorage`，键名是 `web-small-game.progression.v1`。

## 文件说明

- `index.html`
  - 最小 HTML 外壳。
  - 设置 `lang="zh-CN"`。
  - 依次加载 `styles.css`、p5.js CDN、`i18n.js`、`snakeLogic.js`、`progression.js` 和 `sketch.js`。

- `styles.css`
  - 提供全屏深色霓虹视觉风格。
  - 包含径向光晕背景、较轻的扫描线、细网格叠层、固定全屏画布样式，以及居中的 `noscript` 提示。
  - 禁用页面滚动、浏览器触控抢占和文本选中，以保证移动端滑动操作稳定。

- `i18n.js`
  - 集中维护中文和英文文案。
  - 默认语言是中文。
  - 中文文案偏游戏化表达，不做生硬直译。
  - 包含主界面、设置、模式、状态、碰撞提示、覆盖层、成长系统、每日挑战、合约和成就文案。
  - 通过 `WebSmallGameI18N` 暴露给渲染层。

- `snakeLogic.js`
  - 游戏状态与规则引擎。
  - 使用 UMD 风格包装：浏览器中挂到 `globalThis/window.SnakeLogic`，Node 环境支持 `module.exports`。
  - 负责网格规则、移动、碰撞、计分、食物生成、模式参数、墙体、传送门和步进速度。
  - 不承载界面标题或描述文案，玩法逻辑和语言内容保持分离。

- `progression.js`
  - 本地成长系统。
  - 使用 UMD 风格包装：浏览器中挂到 `WebSmallGameProgression`，Node 环境也可单独测试。
  - 负责读取/保存进度、创建每日挑战、生成 seeded random、记录单局目标、完成合约、解锁成就和发放芯片奖励。

- `sketch.js`
  - p5.js 展示层与交互层。
  - 负责画布初始化、响应式布局、首页、成就页、游戏渲染、视觉反馈、音效、震动、键盘输入、鼠标/触控输入、暂停、重开、模式切换和成长系统接入。
  - 从 `i18n.js` 读取中英文文案。
  - 从 `progression.js` 读取成长档案、每日挑战和奖励结果。
  - 关键霓虹文字使用 Canvas 原生 `strokeText/fillText` 辅助绘制，包含深色外轮廓、低眩光霓虹描边和更大的可读字号。

## 运行方式

- 入口文件：`index.html`
- 框架/库：p5.js `1.9.4`，通过 jsDelivr CDN 加载
- 网格尺寸：`24 x 24`
- 当前版本不需要本地服务器，直接打开 `index.html` 即可运行。
- 默认界面语言是中文。
- 语言切换入口在首页的设置抽屉里，也可以按键盘 `L` 切换。

## 首页结构

首页不是普通模式列表，而是更偏“游戏大厅”的结构：

- 顶部是霓虹标题 `WebSmallGame` 和副标题。
- 默认展示 `经典模式` 和 `每日挑战`。
- `更多模式` 是抽屉按钮，用来展开/收起 `霓虹疾行`、`电路迷宫`、`传送漂移`。
- `成长档案` 卡片展示芯片数和成就解锁进度，点击后进入成就墙。
- `设置` 是抽屉按钮，目前里面放语言切换。

## 游戏模式

- `经典模式`
  - 熟悉的节奏，考验每一次转弯。
  - 移动速度恒定。

- `霓虹疾行`
  - 连吃越快，节奏越狠。
  - 每吃 3 个食物速度提升一次。
  - 快速连续吃到食物时启用连击计分。
  - 单个食物基础分为 10，连击等级每提升一级额外增加 5 分。

- `电路迷宫`
  - 电墙封住去路，绕行才是本事。
  - 当前电墙数量为 22。
  - 撞到电墙会触发专属电击反馈。

- `传送漂移`
  - 传送门会突然改写玩家路线。
  - 进入一个传送门会从另一个传送门出来。
  - 每吃 5 个食物，传送门会重新迁移。

## 成长系统

`progression.js` 维护的档案字段包括：

- `chips`
- `runs`
- `bestScoreByMode`
- `achievements`
- `contractCompletions`
- `playedModes`
- `dailyClears`

奖励规则：

- 普通合约奖励 `1` 芯片。
- 成就解锁奖励 `3` 芯片。
- 每日挑战完成奖励 `5` 芯片。

每个模式有一组霓虹合约，例如分数、食物、存活 tick、连击、传送次数等目标。
成就包括首次吃食物、100 分、250 分、疾行连击、电墙行者、传送老手、四模式点亮、不暂停 120 分、首次每日、累计 10 个合约和清空棋盘。

每日挑战根据本地日期生成：

- 经典模式：目标分数 140
- 霓虹疾行：目标连击 x6
- 电路迷宫：目标分数 110
- 传送漂移：目标传送 5 次

每日挑战使用 seeded random，让当天同一挑战有可复现的随机序列。

## 操作方式

- 选择经典模式：点击经典模式卡片，或按 `1`。
- 展开更多模式：点击 `更多模式`。
- 选择其他模式：展开后点击模式卡片，或按 `2` 到 `4`。
- 每日挑战：点击每日挑战卡，或按 `D`。
- 成长档案/成就墙：点击成长档案卡，或按 `A`。
- 移动：方向键或 `WASD`。
- 移动端：当前不显示屏幕方向键，主要通过画布滑动转向。
- 移动端覆盖层：游戏结束/胜利时会显示可点的模式返回按钮。
- 暂停/继续：`Space` 或 `P`。
- 重开：`R` 或 `Enter`；游戏结束时点击画布也会重开。
- 返回模式选择：`M`。
- 切换语言：设置抽屉里的语言行，或按 `L`。
- 开关音效：`V`。
- 切换视觉反馈强度：`F`。

## 移动端输入与布局

- 使用 `shouldUseTouchLayout()` 判断移动触控布局。
- 判定综合考虑粗指针、无 hover、用户代理、触点数量和视口尺寸。
- `shouldShowTouchControls()` 当前固定返回 `false`，所以屏幕方向键不会显示。
- 移动端支持滑动转向，阈值由 `SWIPE_MIN_DISTANCE` 和 `SWIPE_DOMINANCE` 控制。
- 滑动成功后会更新起点，因此连续滑动可以连续变向。
- 移动端棋盘布局减少底部保留区，不再为方向键预留大块空间。

## 渲染与反馈

整体视觉是高对比度霓虹朋克风：

- 酸性绿色蛇身
- 粉色食物
- 青色/粉色棋盘边框
- 动态背景线条
- 较轻的 CSS 扫描线和细网格叠层
- 冲击环、粒子、故障切片、横幅、命中停顿、屏幕震动和全屏闪光
- 主标题和游戏结束/暂停/胜利标题使用霓虹字效
- 关键文字带深色外轮廓，避免亮光把字形糊掉
- 覆盖层中的分数、继续、返回模式等文本使用可读霓虹文本函数绘制

音效由 Web Audio API 在用户交互后生成。
可用时还会调用 `navigator.vibrate` 触发震动反馈。

反馈模式分为：

- `full`
  - 完整视觉反馈、命中停顿、屏幕震动、震动反馈和更明显的音效。

- `reduced`
  - 减少粒子、不显示全屏冲击事件，降低震动/停顿强度、音量，并在切入该模式时取消震动。

## 重要实现细节

- `snakeLogic.js` 不依赖 p5.js，并且支持 Node 环境导出。
- `progression.js` 也支持 Node 环境导出，方便后续单独测试成长逻辑。
- 方向输入会进入队列，最多保留 4 个输入，并拒绝反向掉头。
- 蛇没有增长时，允许移动到上一帧尾巴所在格子。
- 食物生成会避开蛇身、墙体和传送门。
- 迷宫墙体和传送门都会避开开局保护区域。
- 传送门移动会先于碰撞和食物检测计算，食物检测使用传送后的目标格。
- 如果没有空格可以生成食物，游戏会进入 `status: "won"`。
- 暂停会通过 `Progression.markPaused()` 标记当前 run，影响“不暂停”类成就。
- 每局结束时通过 `Progression.finalizeRun()` 写入局数、最高分、成就和每日完成记录。
