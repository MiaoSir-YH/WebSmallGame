# WebSmallGame 项目记忆归档

## 当前快照

`WebSmallGame` 是一个静态网页小游戏合集项目。
当前主页是小游戏入口大厅，现阶段已接入四个游戏：霓虹贪吃蛇、霓虹打砖块、电路闪避、记忆矩阵。
项目使用 p5.js 和原生 JavaScript 编写，没有包管理器、构建工具或本地服务依赖。

当前版本里，贪吃蛇已经不只是单局小游戏，而是带有 8 种模式和本地成长系统：

- 霓虹合约
- 成就墙
- 每日挑战
- 芯片奖励
- 各模式最高分

成长进度保存在浏览器 `localStorage`，键名是 `web-small-game.progression.v1`。

## 当前分支状态

- 当前工作区以 `origin/main` 的小游戏合集版本为基线。
- 本地 `main` 的提交指针仍比 `origin/main` 旧；判断项目现状时以当前工作区文件和 `origin/main` 为准。
- 本轮改动是在远程小游戏大厅架构上扩展贪吃蛇 8 模式，而不是回退到旧单页贪吃蛇。

## 文件说明

- `index.html`
  - 最小 HTML 外壳。
  - 设置 `lang="zh-CN"`。
  - 依次加载 `styles.css`、p5.js CDN、`gameCatalog.js`、`i18n.js`、`snakeLogic.js`、`progression.js`、`miniGames.js` 和 `sketch.js`。

- `styles.css`
  - 提供全屏深色霓虹视觉风格。
  - 包含径向光晕背景、较轻的扫描线、细网格叠层、固定全屏画布样式，以及居中的 `noscript` 提示。
  - 禁用页面滚动、浏览器触控抢占和文本选中，以保证移动端滑动操作稳定。

- `i18n.js`
  - 集中维护中文和英文文案。
  - 默认语言是中文。
  - 中文文案偏游戏化表达，不做生硬直译。
  - 包含小游戏入口、贪吃蛇大厅、设置、模式、状态、碰撞提示、覆盖层、成长系统、每日挑战、合约和成就文案。
  - 通过 `WebSmallGameI18N` 暴露给渲染层。

- `gameCatalog.js`
  - 小游戏目录扩展点。
  - 当前包含 `snake`、`breakout`、`dodge`、`memory` 四项，编号为 `1` 到 `4`。
  - 后续新增小游戏时，优先在这里增加目录项，再提供对应小游戏模块。

- `miniGames.js`
  - 轻量小游戏模块集合，当前包含 `breakout`、`dodge`、`memory`。
  - 每个模块暴露 `id`、`createState()`、`reset()`、`update()`、`render()`、`keyPressed()`、`pointerPressed()`、`pointerMoved()`。
  - 新小游戏目前只维护本局分数、胜负状态和自己的输入逻辑，不接入贪吃蛇成长系统。
  - 每个小游戏 state 带 `events` 队列，玩法节点通过事件向 `sketch.js` 汇报命中、拾取、失误、通关等反馈点；事件坐标使用小游戏区域内 0-1 坐标。

- `snakeLogic.js`
  - 游戏状态与规则引擎。
  - 使用 UMD 风格包装：浏览器中挂到 `globalThis/window.SnakeLogic`，Node 环境支持 `module.exports`。
  - 负责网格规则、移动、碰撞、计分、食物生成、模式参数、墙体、传送门和步进速度。
  - 额外支持奖励点、追踪体、坍缩安全区和双核镜像蛇。
  - 不承载界面标题或描述文案，玩法逻辑和语言内容保持分离。

- `progression.js`
  - 本地成长系统。
  - 使用 UMD 风格包装：浏览器中挂到 `WebSmallGameProgression`，Node 环境也可单独测试。
  - 负责读取/保存进度、创建每日挑战、生成 seeded random、记录单局目标、完成合约、解锁成就和发放芯片奖励。

- `sketch.js`
  - p5.js 展示层与交互层。
  - 负责画布初始化、响应式布局、小游戏入口大厅、贪吃蛇大厅、成就页、游戏渲染、视觉反馈、音效、震动、键盘输入、鼠标/触控输入、暂停、重开、模式切换和成长系统接入。
  - 从 `i18n.js` 读取中英文文案。
  - 从 `gameCatalog.js` 读取小游戏入口目录。
  - 从 `miniGames.js` 读取新增小游戏模块，并通过 `miniPlaying` 屏幕状态统一运行。
  - 从 `progression.js` 读取成长档案、每日挑战和奖励结果。
  - 关键霓虹文字使用 Canvas 原生 `strokeText/fillText` 辅助绘制，包含深色外轮廓、低眩光霓虹描边和更大的可读字号。

## 新增轻量小游戏

- `霓虹打砖块`
  - 类 Breakout 玩法，操控挡板反弹能量球击碎砖块。
  - 支持键盘 A/D、方向键、鼠标移动和触控拖动。
  - 三条生命，清空砖块胜利。
  - 反馈点包括挡板反弹、墙体反弹、击碎砖块、丢球扣命和通关。

- `电路闪避`
  - 单屏闪避玩法，移动核心方块躲开电流障碍。
  - 支持 WASD、方向键和触控点击目标移动。
  - 拾取芯片加分，达到 300 分胜利。
  - 反馈点包括拾取芯片、生成电流、撞击失败和 300 分通关。

- `记忆矩阵`
  - 4x4 亮灯记忆玩法。
  - 等待系统播放序列后，按顺序点击矩阵。
  - 支持鼠标、触控和键盘 `1-4`、`QWET`、`ASDG`、`ZXCB`，完成 8 轮胜利。
  - 反馈点包括播放亮灯、点对、点错、进入下一轮和第 8 轮通关。

## 贪吃蛇大厅结构

进入 `霓虹贪吃蛇` 后展示贪吃蛇自己的二级大厅：

- 顶部标题显示 `霓虹贪吃蛇` / `Neon Snake`。
- 左上角 `大厅` 按钮返回小游戏入口主页。
- 默认展示 `经典模式` 和 `每日挑战`。
- `更多模式` 抽屉展开/收起其余 7 个蛇模式；窄屏展开时使用紧凑两列布局。
- `成长档案` 卡片展示芯片数和成就解锁进度，点击后进入成就墙。
- `设置` 抽屉目前包含语言切换。

## 游戏模式

- `经典模式`
  - 熟悉的节奏，考验每一次转弯。
  - 移动速度恒定。

- `霓虹疾行`
  - 连吃越快，节奏越狠。
  - 每吃 3 个食物速度提升一次。
  - 快速连续吃到食物时启用连击计分。
  - 单个食物基础分为 10，连击等级每提升一级额外增加 5 分。

- `电路迷宫`
  - 电墙封住去路，绕行才是本事。
  - 当前电墙数量为 22。
  - 撞到电墙会触发专属电击反馈。

- `传送漂移`
  - 传送门会突然改写玩家路线。
  - 进入一个传送门会从另一个传送门出来。
  - 每吃 5 个食物，传送门会重新迁移。

- `连锁爆点`
  - 吃普通食物后生成 3 个限时奖励点。
  - 奖励点 8 tick 内有效，吃到得 15 分并延续连击反馈。
  - 奖励点不会让蛇增长。

- `猎影追踪`
  - 一个追踪体每 2 tick 朝蛇头靠近。
  - 蛇头撞到追踪体，或追踪体追上蛇身，都会结束。
  - 吃食物会触发更强的追踪冲击反馈。

- `坍缩电网`
  - 每吃 4 个食物，安全区向内收缩 1 格。
  - 最多收缩 5 格。
  - 蛇头进入安全区外会以电网失败结束。

- `双核同步`
  - 主蛇从左侧起步，镜像蛇从右侧起步。
  - 两条蛇同步移动，横向方向互为镜像，上下方向一致。
  - 任一蛇头撞墙、撞身体或撞到另一条蛇都会结束。

## 成长系统

`progression.js` 维护的档案字段包括：

- `chips`
- `runs`
- `bestScoreByMode`
- `achievements`
- `contractCompletions`
- `playedModes`
- `dailyClears`

奖励规则：

- 普通合约奖励 `1` 芯片。
- 成就解锁奖励 `3` 芯片。
- 每日挑战完成奖励 `5` 芯片。

每个模式有一组霓虹合约，例如分数、食物、存活 tick、连击、传送次数等目标。
成就包括首次吃食物、100 分、250 分、疾行连击、电墙行者、传送老手、新模式分数挑战、八模式点亮、不暂停 120 分、首次每日、累计 10 个合约和清空棋盘。

每日挑战根据本地日期生成：

- 经典模式：目标分数 140
- 霓虹疾行：目标连击 x6
- 电路迷宫：目标分数 110
- 传送漂移：目标传送 5 次
- 连锁爆点：目标分数 160
- 猎影追踪：目标分数 130
- 坍缩电网：目标分数 130
- 双核同步：目标分数 100

每日挑战使用 seeded random，让当天同一挑战有可复现的随机序列。

## 操作方式

- 进入贪吃蛇：在主页点击 `霓虹贪吃蛇`，或按 `1`。
- 进入新增小游戏：在主页点击对应卡片，或按 `2` 到 `4`。
- 轻量小游戏返回大厅：按 `H`，或点击左上角 `大厅`。
- 轻量小游戏重开：按 `R` / `Enter`，结束后也可以点击画布重开。
- 返回小游戏大厅：在贪吃蛇大厅点击 `大厅`，或按 `H`。
- 选择经典模式：进入贪吃蛇大厅后点击经典模式卡片，或按 `1`。
- 展开更多模式：点击 `更多模式`。
- 选择其他模式：展开后点击模式卡片，或按 `2` 到 `8`。
- 每日挑战：点击每日挑战卡，或按 `D`。
- 成长档案/成就墙：点击成长档案卡，或按 `A`。
- 移动：方向键或 `WASD`。
- 移动端：当前不显示屏幕方向键，主要通过画布滑动转向。
- 移动端覆盖层：游戏结束/胜利时会显示可点的模式返回按钮。
- 暂停/继续：`Space` 或 `P`。
- 重开：`R` 或 `Enter`；游戏结束时点击画布也会重开。
- 返回模式选择：`M`。
- 切换语言：设置抽屉里的语言行，或按 `L`。
- 开关音效：`V`。
- 切换视觉反馈强度：`F`。

## 移动端输入与布局

- 使用 `shouldUseTouchLayout()` 判断移动触控布局。
- 判定综合考虑粗指针、无 hover、用户代理、触点数量和视口尺寸。
- `shouldShowTouchControls()` 当前固定返回 `false`，所以屏幕方向键不会显示。
- 移动端支持滑动转向，阈值由 `SWIPE_MIN_DISTANCE` 和 `SWIPE_DOMINANCE` 控制。
- 滑动成功后会更新起点，因此连续滑动可以连续变向。
- 移动端棋盘布局减少底部保留区，不再为方向键预留大块空间。

## 渲染与反馈

整体视觉是高对比度霓虹朋克风：

- 酸性绿色蛇身
- 粉色食物
- 青色/粉色棋盘边框
- 动态背景线条
- 较轻的 CSS 扫描线和细网格叠层
- 冲击环、粒子、故障切片、横幅、命中停顿、屏幕震动和全屏闪光
- 主标题和游戏结束/暂停/胜利标题使用霓虹字效
- 关键文字带深色外轮廓，避免亮光把字形糊掉
- 覆盖层中的分数、继续、返回模式等文本使用可读霓虹文本函数绘制
- 新小游戏通过 `miniFeedbacks` 绘制冲击环、粒子爆裂、短暂拖影和分数飘字；强反馈模式下还会叠加全屏故障切片、banner、震动、音效、hit-stop 和屏幕震动。
- 贪吃蛇在棋盘清空胜利时会额外触发强通关反馈，成长系统和奖励 toast 保持原逻辑。

音效由 Web Audio API 在用户交互后生成。
可用时还会调用 `navigator.vibrate` 触发震动反馈。

反馈模式分为：

- `full`
  - 完整视觉反馈、命中停顿、屏幕震动、震动反馈和更明显的音效。

- `reduced`
  - 减少粒子、不显示全屏冲击事件，降低震动/停顿强度、音量，并在切入该模式时取消震动。

## 重要实现细节

- `snakeLogic.js` 不依赖 p5.js，并且支持 Node 环境导出。
- `progression.js` 也支持 Node 环境导出，方便后续单独测试成长逻辑。
- 方向输入会进入队列，最多保留 4 个输入，并拒绝反向掉头。
- 蛇没有增长时，允许移动到上一帧尾巴所在格子。
- 食物生成会避开蛇身、墙体和传送门。
- 迷宫墙体和传送门都会避开开局保护区域。
- 传送门移动会先于碰撞和食物检测计算，食物检测使用传送后的目标格。
- 食物生成会避开奖励点、追踪体、坍缩安全区外和镜像蛇。
- 如果没有空格可以生成食物，游戏会进入 `status: "won"`。
- `progression.js` 的模式序列已同步到 8 个贪吃蛇模式，最高分、合约、每日挑战和八模式成就都会识别新模式。
- 暂停会通过 `Progression.markPaused()` 标记当前 run，影响“不暂停”类成就。
- 每局结束时通过 `Progression.finalizeRun()` 写入局数、最高分、成就和每日完成记录。
